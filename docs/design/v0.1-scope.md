# v0.1 Scope

## 목표
6주 안에 production-grade MVP 출시. Go 서비스에 import 해서 쓰는 라이브러리 모드 (HTTP proxy 모드는 v0.2).

## In Scope (반드시 포함)

### 1. Provider Abstraction
- OpenAI Chat Completions API
- Anthropic Messages API
- 통일된 인터페이스
  - `Provider.Chat(ctx, ChatRequest) (ChatResponse, error)`
  - `Provider.Name() string`
  - `Provider.SupportsModel(model string) bool`

### 2. Failover Routing
- Primary → Fallback 순서 지정
- Trigger: timeout, 5xx, rate-limit 응답
- 고루틴 + context propagation 기반
- Failover 시 metric 기록 (어디서 어디로, 왜)

### 3. Distributed Rate Limit
- Redis 백엔드
- RPM (requests per minute) + TPM (tokens per minute) 동시 적용
- Sliding window 알고리즘
- Lua script로 원자성 확보
- 분산락 패턴 (멀티 인스턴스 환경)

### 4. Observability
**Metrics (Prometheus):**
- `llm_request_total{provider, model, status}`
- `llm_request_duration_seconds{provider, model}`
- `llm_tokens_total{provider, model, type=input|output}`
- `llm_cost_usd_total{provider, model}`
- `llm_failover_total{from, to, reason}`

**Structured logging (slog):**
- Request ID propagation (context에서)
- Provider 선택 trace
- Latency breakdown (provider call vs gateway overhead)

### 5. Documentation
- README (영문, RDD)
- ADR 6편 이상 (한국어 우선)
- Architecture diagram (mermaid 또는 excalidraw)
- examples/ 디렉토리 (3-5개 시나리오)

## Out of Scope (v0.2+)
| 항목 | 이유 |
|---|---|
| Semantic cache | 정확도 리스크 + 사용 사례 좁음. 벤더 prompt caching이 더 안전. |
| Streaming responses | 구현 복잡도 ↑, v0.1 검증 후 추가. |
| Embedding / Image / Audio | 사용 사례 다양, v0.2에서 모달리티별 추가. |
| Gemini / Bedrock / Vertex | 두 벤더로 패턴 검증 후 확장. |
| HTTP proxy 모드 | 라이브러리 모드 정착 후 추가. |
| Cost budget 가드 | rate limit 정착 후 위에 쌓기. |
| A/B 라우팅 | 핵심 가치 아님. |

## 모듈 경계 (단방향 의존성)
```
gateway/                    # 진입점: Config, New(), Chat()
├── pkg/
│   ├── provider/           # 벤더 추상화 (gateway는 이걸만 의존)
│   │   ├── openai/
│   │   └── anthropic/
│   ├── router/             # failover 라우팅
│   ├── ratelimit/          # 분산 rate limit
│   ├── metrics/            # Prometheus
│   └── logging/            # slog
├── internal/
│   └── testutil/           # recorded responses, mock Redis 등
├── examples/               # 사용 예시
└── docs/                   # 본 문서들
```

**원칙:** 각 `pkg/*`는 다른 `pkg/*`를 모를 수 있어야 한다. 합성은 최상위 `gateway` 패키지에서만.

## 비스코프 결정 트레일
- "v0.1에서 HTTP proxy도 같이?" → ADR-001에서 No로 결정
- "v0.1에서 Gemini도?" → 두 벤더로 패턴 검증 먼저
- "Semantic cache는 v0.1에 옵션으로?" → 정확도 리스크 + 매력 감소로 v0.2 이후
